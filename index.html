Joint Space Workspace Splines C-Space

01

Geometry Path & Trajectory

A geometry path describes the locus of the end-effector or joint configuration in parametric form — independent of time. A trajectory attaches a timing law s(t) to that path, giving the robot motion its temporal dimension.

📍

Path in Workspace

The locus of the end-effector expressed as p(s) in Cartesian coordinates, parameterised by curve length s ∈ [0, L].

⚙️

Path in Joint Space

Described as a vector of joint variables q(σ) where each joint variable is a function of parameter σ ∈ [σ_min, σ_max].

⏱️

Timing Law

The function s = s(t) (or σ = σ(t)) maps time to path progress, turning a geometric curve into a trajectory with velocity and acceleration profiles.

🔄

Complete Trajectory

Combining path + timing law gives p(s(t)) or q(σ(t)) — a position function of time with continuous velocity guarantees.

Workspace vs. Joint Space

Aspect Workspace (Cartesian) Joint Space (Config.)
End-effector constraints Natural, easy to specify Require IK transformation
Obstacle avoidance Straightforward for simple environments Better for redundant robots
Joint limits Must be checked afterward Directly enforced
Singularity handling Challenging Can be designed around
Control implementation Needs IK at runtime Direct motor commands
01.2

Configuration Space (C-Space)

The configuration space (C-space) is a multidimensional space where each dimension corresponds to one degree of freedom (one joint). A single point in C-space represents a complete robot configuration.

ℹ️

For an n-DOF robot, the C-space is n-dimensional. A 2-DOF planar robot has a 2D C-space; the full configuration is given by the pair (q₁, q₂).

Path Planning in C-Space

Planning algorithms sample points in C-space and connect them into feasible motion paths. Obstacles in physical space become regions in C-space — C-obstacles — that the path must avoid.

2-DOF Robot C-Space Representation

Joint constraints example:

Joint 1: 0° ≤ q₁ ≤ 180° Joint 2: 0° ≤ q₂ ≤ 360° // Each (q₁,q₂) pair is a point // in 2D configuration space

Path planning goal:

Start configuration

Point q_A in C-space

Avoid C-obstacles

Regions blocked by physical obstacles

Goal configuration

Point q_B in C-space

02

Trajectory Planning in Joint Space

Joint-space trajectories describe how each joint variable evolves over time, rather than specifying the end-effector path directly. This approach is natural for low-level motor control.

📏

Linear Trajectory

Each joint follows a straight line from initial to target configuration. Simple but may violate velocity continuity at waypoints.

〰️

Spline Trajectory

Cubic or quintic splines interpolate smoothly between configurations, ensuring continuous position, velocity, and optionally acceleration.

🎯

Point-to-Point (PTP)

Moves from one joint configuration to another — the workhorse of pick-and-place and manipulation tasks.

02.2

Point-to-Point Polynomial Trajectories

Cubic Polynomial (3rd Order)

Four boundary conditions (position and velocity at start and end) uniquely determine a cubic polynomial. Given tᵢ, tᶠ, qᵢ, qᶠ, q̇ᵢ, q̇ᶠ:

Position function
q(t) = a₃(t-tᵢ)³ + a₂(t-tᵢ)² + a₁(t-tᵢ) + a₀
Matrix system [ h = tᶠ − tᵢ ]
⎡ 1 0 0 0 ⎤ ⎡a₀⎤ ⎡ qᵢ ⎤ ⎢ 0 1 0 0 ⎥ ⎢a₁⎥ = ⎢ q̇ᵢ ⎥ ⎢ 1 h h² h³ ⎥ ⎢a₂⎥ ⎢ qᶠ ⎥ ⎣ 0 1 2h 3h² ⎦ ⎣a₃⎦ ⎣ q̇ᶠ ⎦
Worked Example 1 — Zero velocity endpoints

Boundary Conditions

tᵢ=0, q(0)=10°, q̇(0)=0 tᶠ=4, q(4)=45°, q̇(4)=0
// Solution vector: a = [10.00, 0, 6.5625, −1.0938]ᵀ // Resulting trajectory: q(t) = 10 + 6.5625 t² − 1.0938 t³

Quintic Polynomial (5th Order)

Six boundary conditions — position, velocity, and acceleration at both endpoints — determine a quintic. This ensures smoother motion with no acceleration discontinuities.

Position function
q(t) = a₅(t-tᵢ)⁵ + a₄(t-tᵢ)⁴ + a₃(t-tᵢ)³ + a₂(t-tᵢ)² + a₁(t-tᵢ) + a₀
6×6 Matrix system [ h = tᶠ − tᵢ ]
⎡1 0 0 0 0 0 ⎤ ⎡a₀⎤ ⎡ qᵢ ⎤ ⎢0 1 0 0 0 0 ⎥ ⎢a₁⎥ ⎢ q̇ᵢ ⎥ ⎢0 0 2 0 0 0 ⎥ ⎢a₂⎥ = ⎢ q̈ᵢ ⎥ ⎢1 h h² h³ h⁴ h⁵⎥ ⎢a₃⎥ ⎢ qᶠ ⎥ ⎢0 1 2h 3h² 4h³ 5h⁴⎥ ⎢a₄⎥ ⎢ q̇ᶠ ⎥ ⎣0 0 2 6h 12h² 20h³⎦ ⎣a₅⎦ ⎣ q̈ᶠ ⎦
💡

Quintic polynomials are preferred when smooth, jerk-limited motion is critical — e.g., in medical robotics, painting, or precision assembly.

02.3

Multi-Phase (Trapezoidal) Trajectories

An industrial favourite: a piecewise trajectory combining constant-acceleration, constant-velocity, and constant-deceleration phases — giving a trapezoidal velocity profile.

Isosceles Trapezoidal Velocity Profile

Accel
[0, tₐ]
Constant Velocity
[tₐ, tᶠ−tₐ]
Decel
[tᶠ−tₐ, tᶠ]
t = 0t = tₐt = tᶠ − tₐt = tᶠ

Design Equations

Feasibility condition on acceleration
// Minimum required acceleration: a ≥ 4|qᶠ − qᵢ| / tᶠ² // Acceleration time: tₐ = tᶠ/2 − √(tᶠ²/4 − (qᶠ−qᵢ)/a) / 2 // Peak velocity: vₘₐₓ = a · tₐ
Worked Example — Trapezoidal profile

Given: q(0)=0°, q(4s)=30°, aₘₐₓ=10°/s²

// Check feasibility: 4|30-0|/16 = 7.5 < 10 ✓ // Acceleration time: tₐ = 1 s → vₘₐₓ = 10 °/s Accel [0,1]: q(t) = 5t² Const [1,3]: q(t) = 10t − 5 Decel [3,4]: q(t) = −5t² + 40t − 50
ℹ️

Other multi-phase profiles include isosceles trapezoidal acceleration (smoother velocity transitions) and sinusoidal acceleration (zero jerk at phase boundaries).

03

Trajectory Planning in Workspace

Workspace trajectory design plans paths for the end-effector directly in Cartesian space. The same polynomial/spline techniques apply, but with Cartesian coordinates (x, y, z) and orientation.

📐

Point-to-Point

End-effector moves directly from start to goal. Solved by applying IK at both ends and designing joint trajectories.

➡️

Linear Trajectory

End-effector tracks a straight-line path. Parameterised by arc length s with a separate timing law.

🔵

Circular Trajectory

End-effector follows a circular arc described in a local frame attached to the circle centre.

🛤️

Spline / Waypoints

Smooth curves through a set of intermediate waypoints — ideal for obstacle avoidance or complex part geometries.

⚠️

Workspace trajectories require running inverse kinematics at each discretisation step — this increases computational cost. Interpolation between discrete points introduces position errors whose magnitude depends on step resolution and algorithm choice.

03.2

End-Effector Position Trajectories

Linear Trajectory

The parametric line from p_A to p_B (length L) is:

p(s) = p_A + (s/L)(p_B − p_A) // s ∈ [0, L], velocity: ṗ = (p_B − p_A)/L · ṡ // Choose s(t) as cubic polynomial for v-boundary conditions: s(tₐ) = 0, ṡ(tₐ) = vₐ s(tʙ) = L, ṡ(tʙ) = vʙ
Example — Linear path A(1,1,0) → B(4,5,0), T=4s
// Direction unit vector: û = [3, 4, 0]ᵀ / 5 // Cubic timing law s(t) with ṡ(0)=0, ṡ(4)=5: s(t) = −0.1563 t³ + 0.3125 t² x(t) = 1 + (3/5)·s(t) y(t) = 1 + (4/5)·s(t)

Circular Trajectory

The end-effector traces an arc of radius r, centre C, from angle θ₀ to θᶠ in a local frame.

p_M(s) = p_C + R⁰₁ · u_M(s) // Parametric position on arc: u_M(s) = r·[ cos(θ₀ + s/r), sin(θ₀ + s/r), 0 ]ᵀ // Velocity and acceleration: ṗ_M = (dp_M/ds)·ṡ = t·ṡ p̈_M = t·s̈ + (ṡ²/r)·n
03.3

End-Effector Orientation Trajectories

Orientation can be parameterised via Euler angles, RPY angles, or axis-angle representation. Independent polynomial interpolation is applied to each angle.

Linear Euler-angle interpolation
φ(s(t)) = φᵢ + s(t)·(φᶠ − φᵢ)/‖Δφ‖ ϑ(s(t)) = ϑᵢ + s(t)·(ϑᶠ − ϑᵢ)/‖Δϑ‖ ψ(s(t)) = ψᵢ + s(t)·(ψᶠ − ψᵢ)/‖Δψ‖
ℹ️

At minimum, angular velocity continuity must be ensured. More demanding applications require angular acceleration continuity as well.

04

Concatenated Trajectory Planning

A concatenated trajectory joins multiple trajectory segments end-to-end into a seamless motion sequence — enabling complex multi-step tasks from modular building blocks.

🧩

Modularity

Each segment is designed independently; segments combine into complex motion sequences.

🔗

Smooth Transitions

Carefully designed junction conditions ensure continuity of position, velocity, and acceleration.

🎛️

Flexibility

Segments can have different profiles: linear, circular, polynomial, trapezoidal, etc.

04.2

Waypoint Trajectory — Velocity Continuity

Using N cubic polynomials for N segments, boundary conditions at each waypoint enforce position and velocity continuity (C¹). Acceleration may be discontinuous.

Segment k polynomial (h_k = t_{k+1} − t_k)
q_k(t) = a₃ₖ(t−tₖ)³ + a₂ₖ(t−tₖ)² + a₁ₖ(t−tₖ) + a₀ₖ // 4 conditions per segment: q_k(tₖ) = qₖ q̇_k(tₖ) = q̇ₖ q_k(tₖ₊₁) = qₖ₊₁ q̇_k(tₖ₊₁) = q̇ₖ₊₁
Example — 5 waypoints, prescribed velocities
t = [ 0 2 4 6 8 10 ] q = [-10 20 0 30 40 40 ] q̇ = [ 0 10 -10 10 0 0 ] // Result: C¹ continuity (pos + vel), C⁰ acceleration jumps at nodes
04.3

Waypoint Trajectory — Acceleration Continuity

Approach A: Prescribed Derivatives

Use N quintic polynomials with position, velocity, and acceleration prescribed at every waypoint — guaranteed C² continuity.

Approach B: Natural / Clamped Splines

Use N cubic splines with continuity up to the second derivative enforced at interior nodes, plus two extra conditions at the endpoints.

End condition
Natural Spline

q̈(t₀) = q̈(tₙ) = 0 — zero curvature at endpoints

End condition
Clamped Spline

q̇(t₀), q̇(tₙ) prescribed — controlled endpoint slopes

End condition
Not-A-Knot

3rd derivative matches at 1st and (N−1)th interior knot

Reduced System for Natural Spline

Setting m_k = 2c_k (second-derivative values at nodes), the tridiagonal system for interior nodes becomes:

h_{k-1} m_{k-1} + 2(h_{k-1}+h_k) m_k + h_k m_{k+1} = 6[(q_{k+1}-q_k)/h_k − (q_k-q_{k-1})/h_{k-1}] // k = 1, 2, ..., N-1 (N−1 equations) // Boundary: m₀ = 0, m_N = 0 (natural spline) // → tridiagonal system, efficient O(N) solver
Example — Natural spline through 6 data points

Input data table

i : 0 1 2 3 4 5 t : 4.00 4.35 4.57 4.76 5.26 5.88 q : 4.19 5.77 6.57 6.23 4.90 4.77 h : 0.35 0.22 0.19 0.50 0.62

Five cubic segments are fitted to guarantee C² continuity throughout. The natural spline adds q̈=0 at both endpoints.

💡

The tridiagonal nature of the spline system enables an efficient O(N) LU factorisation — making spline trajectory planning fast even for long sequences of waypoints.
